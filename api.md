## API接口
### 获取视频信息
接口地址：    <font color="cyan">/query/getInfo</font>

验证方式：    无
___
请求方式：GET

请求参数：

| 参数名       | 类型  | 内容    | 必要性 | 备注                                |
|-----------|-----|-------|-----|-----------------------------------|
| bv        | str | BV号   | 必要  | 是以"BV"为起始的一个字符串，<br>为Bilibili视频编号 |
| Timeorder | num | 时间轴排序 | 非必要 | 0时间升序排列<br/>1为时间轴倒序 <br/>默认为0     |

回复正文（json）：

| 参数名     | 类型  | 内容     | 备注                   |                   
|---------|-----|--------|----------------------|
| code    | num | 响应状态   | 200:请求成功<br/>其他：请求失败 |
| message | str | 信息     | 无                    |
| data    | obj | 视频信息对象 | 无                    |

data字段：

| 参数名         | 类型   | 内容     | 备注                           |                   
|-------------|------|--------|------------------------------|
| order       | num  | 时间轴排序  | 0为时间轴正序<br/>1为时间轴倒序<br/>默认为0 |
| video_stats | list | 视频信息列表 | 排序方式依order字段而定               |
| bv          | str  | 视频BV号  | 无                            |
| title       | str  | 视频标题   | 无                            |
| pubdate     | str  | 发布日期   | 无                            |
| up_mid      | num  | 视频UP主  | 无                            |

video_info列表字段：

| 参数名       | 类型  | 内容     | 备注         |                   
|-----------|-----|--------|------------|
| stat_time | num | 状态时间   | 当前视频信息的时间戳 |
| views     | num | 播放量    | 无          |
| likes     | num | 点赞     | 无          |
| coins     | num | 投币数    | 无          |
| reply     | num | 回复数    | 无          |
| online    | num | 在线观看人数 | 无          |

### 手动更新数据
接口地址：   <font color="cyan">/operate/updateInfo</font>

验证方式：   无
___

请求方式：GET

请求参数：

| 参数名 | 类型  | 内容   | 必要性 | 备注                                       |                   
|-----|-----|------|-----|------------------------------------------|
| opt | str | 更新范围 | 必要  | “global”：全局更新<br/>“vi”：只更新视频数据信息，不更新视频列表 |

回复正文（json）：

| 参数名     | 类型  | 内容   | 备注                   |                   
|---------|-----|------|----------------------|
| code    | num | 响应状态 | 200：请求成功<br/>其他：请求失败 |
| message | str | 信息   | 无                    |

>Tips：该接口有访问评率限制，30s内只能执行一次
